# Type Errors That Saved Me From Production Bugs

In multiple projects, I used to treat TypeScript as optional strictness. But once forms and API payloads grew, loose types started causing runtime bugs.

## Common issues I faced

- Wrong payload keys passed to backend endpoints.
- Optional fields not handled before rendering.
- Union states not fully checked in UI logic.

## What changed my workflow

1. Defined API request/response types in one shared location.
2. Used discriminated unions for async status handling.
3. Removed unsafe `any` and replaced with explicit types.
4. Added guard functions for optional/nested fields.

## Outcome

Build-time errors increased a little, but production issues dropped a lot.

The best part: TypeScript started acting like a safety system instead of a blocker.
