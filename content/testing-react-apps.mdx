# How I Tracked a Hard-to-Reproduce Dashboard Bug

In one dashboard, filters randomly reset after navigation. The issue only appeared after a specific flow: apply filter, open item details, then return with browser back.

## Debugging steps that helped

1. Logged state transitions around route changes.
2. Confirmed where filter state was stored and when it was reset.
3. Reproduced the issue with a minimal component.
4. Moved state persistence to URL query params.

## Why it was difficult

- The bug did not happen on direct page loads.
- It depended on browser history behavior.
- UI looked normal, but state source was stale.

## Result

After persisting filters in query params, navigation became stable and predictable for users.

This reminded me that many frontend bugs are state-management bugs disguised as UI issues.
