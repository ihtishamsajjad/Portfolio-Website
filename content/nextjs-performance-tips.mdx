# Why My Portfolio Felt Slow and How I Optimized It

At one point, my portfolio looked good but loaded slowly on mobile. Videos in project cards and large assets increased first render time.

## Performance issues I noticed

- Slow initial paint on low-end phones.
- Heavy media in project cards.
- Unnecessary client-side logic in sections that could stay static.

## What I changed

1. Moved project media to organized local assets.
2. Reduced unnecessary re-renders in UI sections.
3. Kept videos muted, looping, and optimized for inline autoplay.
4. Used cleaner layout composition to reduce complexity.

## Result

The site became smoother on mobile and visually stable when scrolling between sections.

Main lesson: performance work should happen while building, not only at the end before deployment.
