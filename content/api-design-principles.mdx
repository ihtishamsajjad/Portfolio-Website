# How I Fixed Role-Based API Issues in Landora

While building Landora, I faced a difficult bug: agents could access routes meant only for admin users. At first, everything looked correct in middleware, but response handling was inconsistent between routes.

## What went wrong

- Some routes checked only authentication, not role.
- Error responses had different shapes, so frontend handling broke.
- Validation was applied in controllers, but not at route level.

## What solved it

1. Added route-level middleware for each permission boundary.
2. Standardized all API errors into one format.
3. Added request validation before controller execution.
4. Added a minimal permission matrix test suite.

## Result

After this change, admin and agent dashboards became predictable, and frontend error handling got much simpler.

The biggest lesson: auth bugs are rarely one line. They are usually a consistency problem across middleware, validation, and response contracts.
