# UI Inconsistency Problems I Faced Across Project Dashboards

In my client and personal projects, each dashboard evolved separately. After a few weeks, buttons, cards, and form inputs looked inconsistent and made the product feel unpolished.

## Difficulties I faced

- Different border radius and spacing rules across pages.
- Repeated custom CSS classes for almost identical components.
- Form errors styled differently in each section.

## What I changed

1. Introduced shared component primitives for cards, buttons, and inputs.
2. Defined spacing and typography tokens and reused them everywhere.
3. Removed one-off utility chains and replaced them with reusable variants.
4. Added a small visual checklist before each release.

## Result

Development became faster, and UI reviews stopped repeating the same feedback about inconsistency. The product started to feel like one system, not many separate pages.

My key lesson: visual consistency is not just design quality, it directly improves developer speed and confidence.
