# Git Mistakes That Broke My Project Build (And How I Fixed Them)

During one sprint, I merged two branches with conflicting component updates and accidentally pushed broken code to main. Build failed and deployment stopped.

## Problems I faced

- Conflicts in shared UI files after parallel feature work.
- Commit messages too vague to quickly locate issue source.
- Pushing without checking local build status.

## What fixed the workflow

1. Feature branches only, no direct pushes to main.
2. Small commits with clear scope and message.
3. Pull latest main before opening PR.
4. Build locally before each push.

## Result

Merge conflicts became easier to resolve, and broken builds dropped significantly.

My lesson: Git discipline is not extra process, it is part of shipping stable projects.
